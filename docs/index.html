项目概览

TG Forward 是一个功能强大的消息转发工具，专注于将 Telegram 群组消息实时转发到多个通知渠道。支持钉钉机器人、Bark、HarmonyOS_MeoW 等多种通知方式，让您不错过任何重要消息。

主要优势

  • 支持多种通知渠道，灵活配置
  • 消息持久化存储，防止丢失
  • 完善的监控指标，支持 Prometheus
  • 高性能消息队列，确保可靠性

架构设计

graph TB TG[Telegram Groups] --> Handler[Message Handler] Handler --> Queue[Message Queue] Queue --> Processor[Message Processor] Processor --> DD[DingTalk] Processor --> Bark[Bark] Processor --> HM[HarmonyOS MeoW] Queue -.-> Storage[LevelDB Storage] Processor -.-> Metrics[Prometheus Metrics]

核心组件

  • 消息处理器： 负责接收和解析 Telegram 消息，支持文本、图片等多种消息类型。

  • 消息队列： 使用 LevelDB 实现的持久化队列，确保消息可靠传递。

  • 通知适配器： 支持多种通知渠道，可以灵活扩展新的通知方式。

功能特性

多渠道支持

支持 Telegram、钉钉、Bark、HarmonyOS_MeoW 等多个通知渠道

消息存储

支持消息持久化存储，便于历史记录查询和消息恢复

监控指标

提供丰富的监控指标，支持 Prometheus 集成

配置说明

配置文件示例

telegram:
  token: "your-bot-token"
  chat_ids: [123456789]

dingtalk:
  enabled: true
  webhook_url: "https://oapi.dingtalk.com/robot/send?access_token=xxx"
  secret: "your-secret"
  enable_at: true
  at_mobiles: ["1234567890"]

bark:
  server: "https://api.day.app"
  device_key: "your-device-key"

harmony:
  base_url: "https://api.example.com"
  user_ids: ["user1", "user2"]

queue:
  type: "leveldb"
  path: "./data/queue"

metrics:
  enabled: true
  port: 9090

配置项说明

Telegram 配置

  • token: 机器人的访问令牌
  • chat_ids: 需要监听的群组 ID 列表

钉钉配置

  • enabled: 是否启用钉钉通知
  • webhook_url: 钉钉机器人的 Webhook 地址
  • secret: 钉钉机器人的签名密钥
  • enable_at: 是否启用 @ 功能

Bark 配置

  • server: Bark 服务器地址
  • device_key: 设备密钥

HarmonyOS MeoW 配置

  • base_url: API 基础地址
  • user_ids: 用户 ID 列表
